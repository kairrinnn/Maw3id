**1. [Rule 1 - Bug] Fixed vi.fn() constructor mock using arrow function**

- **Issue:** Plan specified `vi.fn().mockImplementation(() => ({ ... }))` (arrow function) for `GoogleGenAI` class mock. Arrow functions cannot be used as constructors (`new` keyword throws `TypeError: not a constructor`).

- **Verification:** Gemini success test now passes — `new GoogleGenAI()` constructs correctly in test environment
